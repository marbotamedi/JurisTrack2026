## Resumo da Task 6.0

```
ID da Tarefa: 6.0
Nome da Tarefa: Finalização: Dashboards, Sintéticos e Alertas
Contexto PRD: O último sprint do épico de monitoramento visa entregar visibilidade do estado do serviço no Grafana Cloud, garantindo dashboards de Golden Signals, monitoramento sintético de uptime e canais de alerta.
Requisitos Tech Spec: Observabilidade completa com OpenTelemetry/Prometheus exportando para o Grafana Cloud, correlação por `requestId`, monitoramento de latência/throughput e alertas configuráveis no SaaS.
Dependências: Task 2.0 (OpenTelemetry + métricas) deve estar ativa; acesso administrativo ao Grafana Cloud e credenciais OTLP/API.
Objetivos Principais: Criar dashboards consumindo os `juristrack_http_*` e spans existentes; configurar synthetics para health checks da API e dashboard; definir alertas baseados em erro >1% e latência P95 >2s com notificações em canais críticos.
Riscos/Desafios: Necessário acesso ao Grafana Cloud real (credentials/endpoint) e coerência entre métricas instrumentadas e painéis; assegurar que alertas acionáveis não disparam falsos positivos.
```

## Plano de Abordagem

```
1. Catalogar quais métricas e spans já estão disponíveis (histogramas, counters, requestId) e mapear painéis essenciais para os Golden Signals.
2. Definir artefatos que podem ser importados no Grafana Cloud: dashboard JSON, alertas (prometheus/Grafana) e guia para sintéticos, incluindo thresholds e canais.
3. Implementar os arquivos/recursos necessários (scripts, JSON, instruções) para facilitar o deploy dos dashboards e alertas no Grafana Cloud.
```
