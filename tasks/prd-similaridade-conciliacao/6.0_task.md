# 6.0 - Feedback e UX em Tempo Real

## Objetivo
- Prover uma experiência fluida e informativa para o usuário durante o processo de conciliação.

## Escopo / Entregáveis
- Implementação de estados de loading e notificações de sucesso/erro.
- Animações de remoção de card.

## Passos e subtarefas
- 6.1 Adicionar spinner ou texto "Processando..." ao clicar nos botões de ação.
- 6.2 Implementar Toast de notificação para erros de rede ou de negócio.
- 6.3 Garantir que o card seja removido do DOM somente após a confirmação da API.

## Dependências
- 4.0 - UI de Conciliação (Frontend).

## Paralelizável?
- Sim, pode ser feito junto com a 4.0 ou 5.0.

## Critérios de aceite
- O usuário nunca deve ficar sem saber se a ação foi concluída ou se está em progresso.
- Em caso de erro, o card deve permanecer na tela para nova tentativa.

## Testes
- 6.4 Simular latência de rede para validar o estado de loading.
- 6.5 Simular erro de API (ex: 500) e validar se o Toast de erro aparece.

## Notas
- Evitar "flicker" de UI durante a remoção dos itens.
