# 4.0 - UI de Conciliação (Frontend)

## Objetivo
- Implementar a interface que permite ao usuário visualizar os itens pendentes e executar as ações de conciliação.

## Escopo / Entregáveis
- Atualização de `public/js/similaridade.js`.
- Implementação dos cards de conciliação no container de similaridade.

## Passos e subtarefas
- 4.1 Criar função para buscar itens pendentes via `GET /api/similaridade/itens/:uploadId`.
- 4.2 Renderizar cards contendo: Texto da publicação, Score, Datas calculadas e botões [Cadastrar] / [Cancelar].
- 4.3 Implementar listeners para os botões chamando os endpoints da Fase 1.

## Dependências
- 3.0 - Endpoints de Conciliação (Backend).

## Paralelizável?
- Não, depende da API estar pronta para renderizar dados reais.

## Critérios de aceite
- A lista de itens deve ser carregada dinamicamente ao entrar na aba de similaridade.
- Botões devem disparar as chamadas de API corretas.

## Testes
- 4.4 Validar se o `uploadId` está sendo capturado corretamente da URL ou estado global.
- 4.5 Testar o clique no botão [Cadastrar] e verificar se o card "desaparece" após sucesso.

## Notas
- Manter o estilo visual consistente com o restante do sistema (Bootstrap 5).
