# 7.0 - Testes de Integridade e Auditoria

## Objetivo
- Garantir que o sistema é resiliente e que todas as ações críticas deixam rastros para compliance.

## Escopo / Entregáveis
- Relatório de validação final.
- Scripts de teste ou suite de testes manuais documentada.

## Passos e subtarefas
- 7.1 Validar integridade transacional: Falha em `Prazo` = nenhuma `Publicacao` criada.
- 7.2 Validar Auditoria: Todo "Cancelar" deve ter um registro correspondente em `similaridade_descartes_auditoria` com `dados_descartados`.
- 7.3 Validar isolamento por `tenant_id`: Um usuário não pode ver ou conciliar itens de outro tenant.

## Dependências
- 3.0 (Backend) e 4.0 (Frontend).

## Paralelizável?
- Não.

## Critérios de aceite
- 100% de consistência entre os dados conciliados e os registros nas tabelas oficiais.
- Auditoria preenchida corretamente para descartes.

## Testes
- 7.4 Executar conciliação de 10 itens seguidos e verificar se o contador de registros nas tabelas de destino é exatamente 10.
- 7.5 Verificar no banco de dados se a coluna `usuario_id` (se disponível) ou metadados de auditoria estão corretos.

## Notas
- Esta tarefa é o "sign-off" da funcionalidade.
